Sistem Informasi Manajemen Agenda
HAK AKSES :
1) Admin 
Hak akses: 
kelola user (operator, kabid, dll) 
kelola master data (kategori agenda, lokasi, instansi) 
lihat semua agenda  edit/hapus agenda siapa pun 
2) Operator 
Hak akses: input agenda 
upload undangan/lampiran 
edit agenda yang dibuat 
3) Kabid 
Hak akses: 
lihat semua agenda bidang 
approve agenda (opsional tapi keren)
lihat rekap agenda bulanan
tandai agenda selesai / dibatalkan 
4) Staff Bidang 
Hak akses: 
lihat agenda yang sudah disetujui
download undangan 
lihat agenda berdasarkan kategori (rapat, kunjungan, survei lapangan)

FLOW USER:
1️⃣ Flow Login & Otorisasi (Global)
[User]
→ Halaman Login
→ Input Email + Password
→ Sistem Auth
→ Validasi Kredensial & Role
→ Redirect ke Dashboard sesuai Role
Cabang hasil:
- Jika valid → masuk sistem
- Jika tidak valid → tampil pesan error
- Jika akun nonaktif → akses ditolak
[User]
→ Klik Logout
→ Sistem Auth
→ Session dihapus
→ Kembali ke Halaman Login

2️⃣ Flow Admin (Pengelolaan Sistem)
2.1 Kelola User
[Admin]
→ Dashboard Admin
→ Menu User Management
→ Tambah / Edit / Nonaktifkan / Hapus User
→ Sistem Simpan ke Database (users)
Catatan:
- Admin menentukan Role dan Unit/Bidang
- User baru mendapat kredensial awal

2.2 Kelola Master Data
[Admin]
→ Menu Master Data
→ Pilih: Kategori / Lokasi / Instansi / Unit
→ CRUD Data
→ Sistem Update Database master

2.3 Kelola & Monitoring Agenda
[Admin]
→ Menu Semua Agenda
→ Filter (Tanggal / Status / Kategori)
→ Lihat Detail Agenda
→ Edit / Hapus Agenda
→ Sistem Update Database


3️⃣ Flow Operator (Input & Pengelolaan Agenda)
3.1 Buat Agenda Baru
[Operator]
→ Dashboard Operator
→ Menu Buat Agenda
→ Isi Form Agenda
→ Simpan sebagai Draft
→ Status: DRAFT

3.2 Kirim Agenda untuk Persetujuan
[Operator]
→ Buka Agenda Draft
→ Klik "Kirim ke Kabid"
→ Status berubah: PENDING
→ Sistem Kirim Notifikasi ke Kabid

3.3 Edit Agenda
[Operator]
→ Buka Agenda Milik Sendiri
→ Edit Data Agenda
→ Update Database
Batasan:
Hanya bisa edit jika status ≠ APPROVED

3.4 Cetak Agenda
[Operator]
→ Menu Cetak Agenda
→ Pilih Harian / Mingguan
→ Generate PDF
→ Download

4️⃣ Flow Kabid Infrastruktur & Kewilayahan (Approval)
4.1 Lihat Agenda Menunggu Persetujuan
[Kabid]
→ Dashboard Kabid
→ Daftar Agenda PENDING

4.2 Persetujuan Agenda
[Kabid]
→ Klik Detail Agenda
→ Lihat Informasi & Lampiran
→ Pilih:
   - Approve
   - Tolak + Catatan
Hasil:
Approve → Status: APPROVED
Tolak → Status: REJECTED
Sistem mencatat approver + timestamp
Notifikasi dikirim ke Operator & Staff

4.3 Update Status Pasca Kegiatan
[Kabid]
→ Buka Agenda APPROVED
→ Tandai Selesai / Dibatalkan
→ Status: COMPLETED / CANCELLED

5️⃣ Flow Staff Bidang (Konsumsi Informasi)
5.1 Lihat Agenda
[Staff]
→ Dashboard Staff
→ Lihat Agenda APPROVED
→ Filter (Kategori / Tanggal)

5.2 Akses Lampiran
[Staff]
→ Buka Detail Agenda
→ Download Undangan / Lampiran

5.3 Upload Dokumentasi (Opsional)
[Staff]
→ Buka Agenda COMPLETED
→ Upload Dokumentasi
→ Sistem Simpan File & Metadata

5️⃣ Flow Staff Bidang (Konsumsi Informasi)
5.1 Lihat Agenda
[Staff]
→ Dashboard Staff
→ Lihat Agenda APPROVED
→ Filter (Kategori / Tanggal)

5.2 Akses Lampiran
[Staff]
→ Buka Detail Agenda
→ Download Undangan / Lampiran

5.3 Upload Dokumentasi (Opsional)
[Staff]
→ Buka Agenda COMPLETED
→ Upload Dokumentasi
→ Sistem Simpan File & Metadata

FUNCTIONAL REQUIREMENTS
A. Authentication & Authorization
FR-A1
Sistem harus menyediakan fitur login menggunakan email dan password.

FR-A2
Sistem harus menerapkan Role-Based Access Control (RBAC).

Acceptance Criteria:
User dengan role Operator tidak dapat mengakses menu User Management.

B. User Management (Admin)
FR-B1
Admin dapat membuat, mengedit, menonaktifkan, dan menghapus user.

FR-B2
Admin dapat menetapkan role dan unit kerja user.

Acceptance Criteria:
User baru dapat login menggunakan kredensial yang diberikan.

C. Master Data (Admin)
FR-C1
Admin dapat mengelola kategori agenda, lokasi, instansi, dan unit kerja.

Acceptance Criteria:
Data master otomatis muncul pada form input agenda.

D. Manajemen Agenda
FR-D1
Operator dapat membuat agenda dengan data wajib:
- judul
- tanggal
- jam
- kategori
- lokasi / tempat
- lampiran

FR-D2
Sistem harus mendukung status agenda:
- DRAFT
- PENDING
- APPROVED
- REJECTED
- COMPLETED
- CANCELLED

FR-D3
- Operator hanya dapat mengedit agenda yang dibuatnya selama status ≠ APPROVED.

FR-D4
- Admin dapat mengedit dan menghapus semua agenda.

E. Approval Agenda (Kabid)
FR-E1
- Kabid dapat melihat daftar agenda berstatus PENDING.

FR-E2
- Kabid dapat menyetujui atau menolak agenda dengan catatan.

FR-E3
- Sistem mencatat waktu dan user yang melakukan approval.

Acceptance Criteria:
- Setelah approve, status berubah menjadi APPROVED dan notifikasi dikirim.

F. Lampiran & Dokumentasi
FR-F1
Operator dapat mengupload lampiran undangan (PDF/JPG/PNG).

FR-F2
Staff dapat mengupload dokumentasi pasca kegiatan.

Acceptance Criteria:
File dapat diunduh oleh user yang berhak.

G. Notifikasi Sistem
FR-G1
Sistem menampilkan notifikasi internal untuk:
- agenda dikirim
- agenda disetujui
- agenda ditolak

FR-H2
User dapat memfilter agenda berdasarkan kategori dan tanggal.

J. Keamanan Sistem
FR-J1
Password harus disimpan dalam bentuk hash.

FR-J2
Sistem membatasi tipe dan ukuran file upload.

FR-J3
Sistem menolak akses tidak sah (403).

Role yang digunakan:
- Admin
- Operator
- Kabid
- Staff
Contoh Data:

| -- | -------- | ------------------------- |
| 1  | Admin    | Pengelola sistem          |
